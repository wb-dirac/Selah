## 9. 风险与缓解措施

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 本地模型内存占用过高导致 OOM | 高 | 高 | 动态模型卸载，提供模型规格建议，4GB RAM 设备限制最大 7B 模型 |
| iOS 后台任务系统限制过严 | 高 | 中 | 降级为通知提醒方案，引导用户开启通知权限 |
| 第三方 App URL Scheme 变更 | 中 | 低 | 维护 URL Scheme 版本兼容表，支持用户自定义 |
| A2A 协议早期生态不成熟 | 高 | 中 | Phase 3 接入，提前评估协议稳定性，保持接口抽象层 |
| Skill 沙箱被绕过 | 低 | 高 | 安装时 AST 静态扫描，运行时网络域名白名单强制执行，QuickJS 安全审计，扫描规则开源持续更新 |
| Anthropic Skill 标准变更 | 中 | 中 | 保持 Manifest 解析层抽象，与标准版本解耦，监控官方变更并快速跟进 |
| GitHub Gist 同步冲突导致数据丢失 | 中 | 高 | 同步前展示 diff，保留本地快照，Last-write-wins 策略配合用户手动仲裁选项 |
| 云端 API 费用超出用户预期 | 中 | 中 | Token 用量实时显示，费用估算，可设硬性上限 |
| 方言语音识别体验差 | 高 | 中 | 明确标注方言需要云端服务，提供云端 STT 快速配置引导 |
