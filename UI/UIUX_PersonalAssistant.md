## 1. 设计语言与系统

### 1.1 设计理念

**"本地第一，无感智能"**——界面始终让用户感觉自己掌握主控权。智能行为应该感觉像是自然延伸，而不是神秘的黑盒。隐私保护不是限制，而是一种可感知的承诺。

整体风格定位：**工业精密感 + 温暖可亲性**。参考精密仪器的质感（分层、边界清晰、状态明确），同时保持对话的温度感。不是冷漠的工具，也不是过度拟人的玩具。

### 1.6 动效原则

- **有意义的动效**：所有动效必须传递信息或引导注意力，不做纯装饰性炫技
- **方向感**：界面切换保持一致的空间方向（右进 = 深入，左出 = 返回，上出 = 关闭）
- **响应速度感**：输入到响应的视觉反馈 < 100ms，哪怕是占位动画
- **本地 vs 云端感**：本地处理用平静的脉冲动画；云端处理用更活跃的进度指示

```
缓动函数
  --ease-out:     cubic-bezier(0.0, 0.0, 0.2, 1)   // 进入动画
  --ease-in:      cubic-bezier(0.4, 0.0, 1, 1)     // 退出动画
  --ease-spring:  cubic-bezier(0.34, 1.56, 0.64, 1) // 弹性，用于确认/成功

时长规格
  --duration-fast:    120ms   // 微交互（hover、focus）
  --duration-normal:  220ms   // 元素进出
  --duration-slow:    380ms   // 页面切换
  --duration-xslow:   600ms   // 大面积布局变化
```

- 选中态：图标填充色 + 文字加粗，底部 3px 高亮线（--color-primary）

## 3. 对话主界面

### 3.1 对话列表页

```
┌─────────────────────────────────────────┐
│  ← 返回    对话历史            + 新建    │  ← 导航栏
├─────────────────────────────────────────┤
│  🔍  搜索对话...                         │  ← 搜索框（常驻）
├─────────────────────────────────────────┤
│  今天                                   │  ← 分组标题（粘性）
│  ┌─────────────────────────────────────┐│
│  │ 帮我查北京到上海的高铁...     14:32  ││  ← 对话条目
│  │ 共 12 条消息  · 本地 🔒              ││
│  └─────────────────────────────────────┘│
│  ┌─────────────────────────────────────┐│
│  │ 菜单热量计算                  09:11  ││
│  │ 共 5 条消息   · 混合 🔀              ││
│  └─────────────────────────────────────┘│
│  昨天                                   │
│  ...                                   │
└─────────────────────────────────────────┘
```

**对话条目细节**：
- 高度：68pt / 68dp（固定），内有 16px 内边距
- 左侧：无头像（助理是单一身份，无需区分）
- 主文字：对话首条用户消息的截断预览（最多 2 行），字号 15px
- 副文字：消息数量 + 隐私处理方式标记（本地/云端/混合）
- 时间戳：右上角，灰色，小字
- 左滑操作：显示"删除"（红色）和"归档"（灰色）按钮
- 长按：显示上下文菜单（复制链接、导出、删除）

**搜索框行为**：
- 常驻于列表顶部，不随滚动消失
- 搜索范围：对话标题 + 消息内容全文搜索（本地索引）
- 搜索结果即时展示，高亮匹配词
- 空搜索状态：提示"搜索消息内容、工具调用记录..."

**消息气泡设计规范**：

用户消息：
```
背景色: --color-primary (#1A2B4A)
文字色: #FFFFFF
圆角: 16px 16px 4px 16px（右下角缩进，指向用户侧）
最大宽度: 75% 屏幕宽度
对齐: 右对齐
内边距: 12px 16px
```

AI 消息：
```
背景色: --color-bg-surface (#FFFFFF)
边框: 1px solid --color-border
文字色: --color-text-primary
圆角: 4px 16px 16px 16px（左上角缩进，指向 AI 侧）
最大宽度: 90% 屏幕宽度（AI 消息通常更长）
对齐: 左对齐
内边距: 14px 16px
阴影: 层级 1
```

**AI 消息元数据行**（消息底部，小字，灰色）：
```
[时间戳]  [处理方式标记]  [Token 数，可选]  [复制] [重试] […更多]
```

**工具调用状态卡**：
- 折叠态：一行展示「图标 + 工具名 + 状态」，右侧可展开箭头
- 展开态：显示输入参数（脱敏）和输出摘要
- 状态颜色：进行中=蓝色脉冲；成功=绿色；失败=红色
- 多个工具串行：竖向堆叠，有连接线表示依赖关系

**Markdown 渲染规范**：
- 代码块：使用 --color-bg-base 背景，JetBrains Mono 字体，右上角复制按钮
- 表格：自动横向滚动，列宽自适应
- 数学公式：KaTeX 渲染
- 图片：居中显示，点击全屏预览
- 链接：下划线 + --color-primary 色，点击内置浏览器或外部打开（用户可选）

**流式输出视觉**：
- AI 回复流式时，消息末尾显示闪烁光标（`|`，1s 周期）
- 生成式 UI 卡片：骨架屏占位，数据到达后平滑淡入
- 不显示"loading..." 纯文字，用视觉动画代替

## 4. 多模态输入工具栏

### 4.1 默认收起态

```
┌────────────────────────────────────────────┐
│  +   [  输入消息...              ] 🎤  ⬆️  │
└────────────────────────────────────────────┘
  ↑                                      ↑
 展开工具                             发送/语音
```

组件细节：
- 整体高度：52pt，带 Safe Area 兼容
- 输入框：多行自动扩展，最大展开 120pt，超出后内部滚动
- `+` 按钮：点击展开多模态工具栏（抽屉向上弹出）
- `🎤` 按钮：按住说话（普通语音模式）；长按 1.5s 切换至实时对话模式
- `⬆️` 发送：输入内容非空时激活为主色调；空态为灰色禁用
- 输入框左侧：附件缩略图（已添加图片/文件时出现）

### 4.2 工具抽屉展开态

```
┌────────────────────────────────────────────┐
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  │
```

**声波动画状态规范**：

| 状态 | 动画描述 | 颜色 |
|------|----------|------|

| 打断（Barge-in） | 波形快速收缩然后切换为用户波形 | 动画过渡流畅 |

- 半透明黑色背景（rgba(0,0,0,0.6)），毛玻璃效果
- 当前 AI 说话内容用白色显示，字号 16px
- 用户刚说的话用灰色显示，字号 14px
- 字幕自动滚动，始终显示最新内容
- 可点击字幕区收起（降低干扰）

**底部控制栏**：
- 静音：麦克风 icon，激活时显示红色划线
- 暂停：暂停整个对话，转为普通文字模式
- 云端标记：不可交互，仅展示当前使用的服务名称

## 17. 无障碍设计规范

### 17.1 颜色对比度

所有文字与背景的对比度需满足 WCAG AA 标准（最低 4.5:1），关键操作区域达到 AAA（7:1）：

```
主文字 (#1C1A18) 在白底:     对比度 19.1:1  ✅ AAA
次要文字 (#6B6560) 在白底:   对比度  5.9:1  ✅ AA
用户气泡白字在深蓝底:         对比度 12.8:1  ✅ AAA
本地标识绿色 (#2D6A4F):      需检验 —— 仅用于辅助色，不单独传达信息
```

**重要原则**：隐私状态（本地/云端）的区分**不能只依靠颜色**，必须同时有图标和文字标签。

### 17.2 VoiceOver / TalkBack 支持

所有图标按钮必须提供 `accessibilityLabel`：
```
+ 按钮      → "展开多模态工具"
🎤 按钮     → "按住录音，长按切换实时对话模式"
⬆️ 按钮     → "发送消息"
··· 按钮    → "更多操作"
🔒 标记     → "本地处理，数据不离开设备"
☁️ 标记     → "云端处理，消息将发送至 Claude 服务器"
```

生成式 UI 卡片需提供语义化描述：
```
高铁卡片 → "G1次高铁，北京南08:05出发，上海虹桥12:28到达，二等座553元"
```

### 17.3 动态字体支持

所有文字尺寸基于系统动态字体（Dynamic Type / SP 单位），支持用户在系统设置中调整字体大小。最大字体等级下，UI 布局需保证可读性（允许换行，不裁切）。

### 17.4 减少动效支持

检测系统"减少动态效果"开关：
- 开启时：禁用所有非必要过渡动画，保留 opacity 淡入淡出
- 实时语音声波动画：降级为静态波形图

### 17.5 键盘导航（桌面端）

- 所有可交互元素支持 Tab 键聚焦
- 聚焦状态有明显的 focus ring（3px 主色调描边）
- 对话气泡支持方向键导航（上下切换消息）
- 生成式 UI 卡片中的按钮支持 Enter 触发
