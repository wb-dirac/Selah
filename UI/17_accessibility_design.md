## 17. 无障碍设计规范

### 17.1 颜色对比度

所有文字与背景的对比度需满足 WCAG AA 标准（最低 4.5:1），关键操作区域达到 AAA（7:1）：

```
主文字 (#1C1A18) 在白底:     对比度 19.1:1  ✅ AAA
次要文字 (#6B6560) 在白底:   对比度  5.9:1  ✅ AA
用户气泡白字在深蓝底:         对比度 12.8:1  ✅ AAA
本地标识绿色 (#2D6A4F):      需检验 —— 仅用于辅助色，不单独传达信息
```

**重要原则**：隐私状态（本地/云端）的区分**不能只依靠颜色**，必须同时有图标和文字标签。

### 17.2 VoiceOver / TalkBack 支持

所有图标按钮必须提供 `accessibilityLabel`：
```
+ 按钮      → "展开多模态工具"
🎤 按钮     → "按住录音，长按切换实时对话模式"
⬆️ 按钮     → "发送消息"
··· 按钮    → "更多操作"
🔒 标记     → "本地处理，数据不离开设备"
☁️ 标记     → "云端处理，消息将发送至 Claude 服务器"
```

生成式 UI 卡片需提供语义化描述：
```
高铁卡片 → "G1次高铁，北京南08:05出发，上海虹桥12:28到达，二等座553元"
```

### 17.3 动态字体支持

所有文字尺寸基于系统动态字体（Dynamic Type / SP 单位），支持用户在系统设置中调整字体大小。最大字体等级下，UI 布局需保证可读性（允许换行，不裁切）。

### 17.4 减少动效支持

检测系统"减少动态效果"开关：
- 开启时：禁用所有非必要过渡动画，保留 opacity 淡入淡出
- 实时语音声波动画：降级为静态波形图

### 17.5 键盘导航（桌面端）

- 所有可交互元素支持 Tab 键聚焦
- 聚焦状态有明显的 focus ring（3px 主色调描边）
- 对话气泡支持方向键导航（上下切换消息）
- 生成式 UI 卡片中的按钮支持 Enter 触发
